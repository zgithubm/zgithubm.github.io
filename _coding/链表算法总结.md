# 链表算法总结

## 1. 合并两个有序链表
**题型：** 将两个有序链表合并为一个新有序链表。

**思路：** 使用双指针（p1, p2）分别遍历两个链表，比较节点值，将较小值接入新链表。通过虚拟头节点（dummy）简化边界处理。

**关键点：** 类似"拉拉链"的过程，虚拟头节点避免空指针问题。

## 2. 链表的分解
**题型：** 将原链表按特定值（如 x）分割为两个部分（小于 x 和大于等于 x）。

**思路：** 创建两个新链表（小链表和大链表），遍历原链表时将节点按条件接入对应链表，最后连接两个链表。

**关键点：** 断开原链表的节点链接，避免成环；使用虚拟头节点简化操作。

## 3. 合并K个有序链表
**题型：** 将多个有序链表合并为一个有序链表。

**思路：** 使用优先级队列（最小堆）维护当前所有链表的头节点，每次取出最小值接入结果链表。

**关键点：** 时间复杂度优化至 O(Nlogk)（N为总节点数，k为链表条数）。

## 4. 单链表的倒数第k个节点
**题型：** 一次遍历找到倒数第k个节点。

**思路：** 使用快慢指针：
- 快指针 p1 先走k步。
- 慢指针 p2 从头开始，与 p1 同步前进，当 p1 到达末尾时，p2 指向倒数第k个节点。

**扩展应用：** 删除倒数第k个节点时，需找到倒数第k+1个节点，配合虚拟头节点处理边界。

## 5. 单链表的中点
**题型：** 一次遍历找到链表中点。

**思路：** 快慢指针（slow每次1步，fast每次2步），当 fast 到达末尾时，slow 指向中点。

**关键点：** 链表长度为偶数时，返回靠后的节点。

## 6. 判断链表是否包含环及环起点
**题型：** 判断链表是否有环，并找到环的起点。

**思路：**
- 判断有环：快慢指针，若相遇则存在环。
- 找环起点：相遇后，将任一指针重置到头节点，两指针同速前进，再次相遇点即为环起点。

**数学原理：** 相遇时慢指针走k步，快指针走2k步，环起点距头节点为k-m（m为相遇点距环起点的距离）。

## 7. 两个链表是否相交
**题型：** 判断两个链表是否相交，并返回交点。

**思路：**
- **拼接法：** 指针 p1 遍历链表A后继续遍历B，p2 遍历B后继续遍历A，使两指针同时到达交点（或同时到达末尾返回 null）。
- **长度对齐法：** 先计算两链表长度，让长链表的指针先走差值步，再同步前进找交点。

**关键点：** 空间复杂度 O(1)，无需额外存储。

## 通用技巧总结

### 虚拟头节点（dummy）
简化边界处理，尤其适用于创建新链表的场景。

### 快慢指针
用于中点、环检测、倒数第k节点等问题。

### 优先级队列
优化多链表合并问题。

### 断开原链表链接
分解链表时避免成环。
